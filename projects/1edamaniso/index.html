A Brief Intro to Meshes

A mesh is a discretization of physical space. That is to say, it is a collection of small simple shapes, called cells (typically triangles and quadrilaterals in 2D and tetrahedra and hexahedra in 3D), that makeup an object and space. These cells are used to approximate the solution to a set of partial differential equations when performing a physical simulation. We set some boundary and initial conditions and the solution then propagates across the mesh by way of a flux integral, which relates one cell to its neighbouring cells. Using more and smaller cells we can create a mesh that comes closer to perfectly respresenting the real object, but it comes at the cost of more memory usage and slower solve speeds. Therefore, creating a high quality mesh is a balancing act. We want to accurately represent the object(s) in questions and capture the physical phenomena that are present but with as few cells as possible.

There are a number of ways in which meshes can be generated, but the only important one to us is advancing-layer generation. If you’re curious you can read about some other techniques here. Three-dimensional advancing-layer mesh generation starts with a surface triangulation of the object(s) in question. From here the volume mesh is constructed one layer at a time by extruding a child node from each parent node on the surface. Each of the children are then connected to each of the parents and their neighbours which gives us, ideally, hexadhedral cells. This continues until we reach a specified number of layers or distance from the original object. For simple convex objects this process can be completed without any hiccups and all we need to specify is the initial extrusion length and a growth rate. Most of the time it doesn’t work out so nicely. Usually, we have objects with concave areas which results in the mesh bunching up on itself. This can be fixed by eliminating certain nodes from the mesh and filling in the holes that are left, this process is called decimation. Decimation is also used to maintain a certain base aspect ratio for cells as the mesh grows, typically a unit aspect ratio.

One final point to touch on before continuing is how meshes are stored. There are two types of meshes: structured and unstructured. As the names suggest, structured meshes have a structured design where the neighbours of each cell are known inherently based on their index. Unstructured meshes, however, must be accompanied by a connection table detailing which cell neighbours which. This makes unstructured meshes both slower, as each time we must lookup who are neighbours are, and more memory intensive. For that reason structured meshes are preferred if they are feasible. The catch is that for most complex engineered objects they are not feasible. Therefore, unstructured meshes must be used, but their generation has long been a bottleneck in the CFD process, as it is slow and often requires manual intervention. For this reason there is a need for automatic mesh generators that can produce high-quality meshes quickly and reliably.

What Is EDAMAniso3D?

EDAMAniso3D is an extension of EDAM3D (Entire Domain Advancing-Layer Mesh Generator) which was developed as part of GRUMMP by Dr. Ollivier-Gooch and ANSLab. GRUMMP aims to create high-quality mixed-element unstructured meshes for physics simulations. GRUMMP primarily uses Delaunay Tetrahedralization for mesh generation, whereas EDAM is an advancing-layer generator.
