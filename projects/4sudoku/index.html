Sudoku Solver

This is a personal project of mine. I’ve written a couple of Sudoku solvers in Python and come across some pretty cool ones online and one video of a solver that solves Sudokus via camera on the screen in real-time. I thought this would be a good test and I am now trying to figure it out using OpenCV and neural networks. I’ll update my progress here. Read on for a bit more of an explanation and history.

I’ve always loved sitting down and solving Sudokus. There’s something pretty satisfying about watching the empty squares disappear as you make progress, ideally without too many mistakes. As I was first starting to learn to code my dad kept suggesting I should write an algorithm to solve Sudokus, but it wasn’t until more recently that I took his advice. It turned out to be a good way to learn about some useful coding algorithms and has now turned into a slightly bigger project.

Researching the problem I learned about the backtracking algorithm. There are many Sudoku solvers on the internet that use variations of it, so I decided to write my own. It is a simple recursion method that is quite a bit faster than the naive solution of just trying every combination of numbers until to find one that works. My method starts in the first empty square along the first row and finds the first number that works. It then go to the next empty cells—working across rows before moving down—and finds the first number that works. If there are no numbers that satisfy the constraints on that square it backtracks to the previous square and increments to the next valid number. This is repeated until we solve the puzzle. Typically, it finishes in a matter of seconds. The solution time, however, is not related to puzzle difficulty, but to how many empty cells we begin with. I was pleased with this solution that worked and solved all solvable Sudokus but not exactly satisfied as it is very much a solution that you can only run on a computer, humans don’t solve Sudokus by backtracking.

Next I set about writing a solver that solved a Sudoku how I personally would solve one. That is, first, cycle through all the empty squares and figure out all the potential numbers. From here, if there are any cells with just a single potential number we can fill that square, then update the rest. For easier puzzles this can be a sufficient algorithm to solve the whole thing, but we need to be a bit more sophisticated for the harder ones. The next step to add was eliminating any paired potential squares from others in its peers. That is, say two squares on the same row each have two potential numbers, 3 and 4, we can now eliminate 3 and 4 as potential numbers from all other squares on that row. If this leaves us with any cells to be filled we can again propagate the new constraint through the puzzle. This addition usually fills in a few more squares, but is not enough to finish most hard Sudokus. The final step in the algorithm is to make a guess. We guess at a square with the fewest potential numbers and propagate the new constraint through the puzzle. If we get stuck again must make another guess. We will now either solve the puzzle or find a square with no potential numbers, in which case we backtrack to the previous guess and try again. This is a much better algorithm in my view as it is relatable, and usually faster for hard puzzles with few clues.

Again, browsing the internet for other solvers I came across a solver by Peter Norvig which is much more elegant and much faster than mine. I’ve adopted much of what he wrote for my next test which is a solver that I only saw on a short video. This Sudoku solver worked in real-time and solved a Sudoku and displayed the solution as it was shown on the camera of a computer. He created this program using OpenCV and training a neural network to recognize the puzzle, so I thought it would be a good project to try it myself. I am now learning OpenCV and hoping to train a neural network along with the help of Novrig’s solver to create my own live solver. If you’re curious you can follow my slow progress on Github.
